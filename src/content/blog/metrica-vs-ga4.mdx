В интернете полно сравнений Метрики и GA4, написанных людьми, которые ни тем ни другим толком не пользовались. Обычно это пересказ документации с выводом «зависит от задач». Спасибо, очень полезно.

Я работаю с обоими инструментами каждый день, настраиваю для клиентов и строю на их данных отчёты. Вот честное сравнение без маркетинговой воды.

## Где Метрика сильнее

### Вебвизор — убийственная фича

Вебвизор — это запись сессий пользователей. Ты буквально смотришь, как человек водил мышкой, куда кликал, где застрял. GA4 ничего подобного не имеет — тебе придётся покупать Hotjar или Clarity (бесплатный, но от Microsoft).

**Когда это критично:** отладка форм, UX-проблемы, анализ поведения на новых страницах. Один час в Вебвизоре часто заменяет неделю A/B-тестирования.

### Карты кликов, скроллинга и ссылок

Тепловые карты прямо в Метрике, без сторонних сервисов. Открыл страницу, включил карту — видишь, куда кликают, до куда скроллят. В GA4 этого нет вообще.

### Простота настройки

Поставил счётчик — он уже работает. Автоматически отслеживает скачивания файлов, внешние ссылки, показы форм. В GA4 для базовых вещей нужно настраивать события, GTM, триггеры... Порог входа в разы выше.

### Российский рынок

Если твой бизнес в России, Метрика — must have. Интеграция с Яндекс.Директ, DataLens, Яндекс.Аудиториями — это экосистема. GA4 с российскими рекламными платформами не дружит.

### Данные без семплирования

Метрика показывает 100% данных в стандартных отчётах. GA4 в бесплатной версии начинает семплировать при больших объёмах, и ты видишь надпись «Based on 42% of sessions». Это значит, что твои цифры — приблизительные.

## Где GA4 сильнее

### BigQuery экспорт

Это главный козырь GA4. Бесплатный экспорт сырых данных в BigQuery. Каждое событие, каждый параметр — всё доступно для SQL-запросов. Ты можешь строить любые отчёты, которые невозможны в интерфейсе.

Метрика тоже экспортирует данные через Logs API, но это сложнее в настройке, и объёмы ограничены. Для полноценной работы с сырыми данными нужен ClickHouse и самостоятельная настройка пайплайна.

### Кросс-платформенная аналитика

GA4 проектировался для мира, где пользователь начинает на телефоне, продолжает на ноутбуке и покупает через приложение. Модель на основе событий и User-ID позволяет склеивать путь пользователя через устройства и платформы.

Метрика тоже умеет Client-ID, но кросс-девайс сопоставление и объединение веба с приложением — это пока слабое место.

### Машинное обучение из коробки

Predictive audiences в GA4 — это не маркетинговый трюк. Модель реально определяет пользователей с высокой вероятностью покупки или оттока, и ты можешь сразу создать аудиторию для ретаргетинга. Метрика ничего подобного не предлагает.

### Exploration reports

Да, интерфейс GA4 — боль. Но раздел Explorations мощный: свободные таблицы, воронки, сегментное перекрытие, path analysis. Это почти BI-инструмент внутри аналитики. В Метрике возможности кастомных отчётов скромнее.

### Глобальная экосистема

Если работаешь с Google Ads, GA4 — нативная интеграция. Данные о конверсиях автоматически передаются в рекламный кабинет, Smart Bidding работает точнее. Плюс интеграция с Looker Studio, Google Sheets и вообще всем Google-стеком.

## Что оба делают плохо

### Атрибуция

Оба инструмента по-прежнему не решают главную проблему: мультиканальная атрибуция остаётся неточной. Last-click доминирует, модели на основе данных (data-driven) требуют больших объёмов, а кросс-девайс сценарии ломают любую модель.

**Реальность:** если хочешь честную атрибуцию, тебе нужно собирать сырые данные и строить свою модель. Ни Метрика, ни GA4 это из коробки не дадут.

### Работа с оффлайн-конверсиями

Клиент увидел рекламу, позвонил по телефону, купил в магазине. Оба инструмента с трудом связывают онлайн-визит с оффлайн-покупкой. Можно загружать данные вручную (Measurement Protocol в GA4, оффлайн-конверсии в Метрике), но это требует серьёзной инфраструктуры.

### Согласие и приватность

Мир движется к cookieless-будущему. GA4 теряет данные из-за блокировщиков (AdBlock режет Google Analytics агрессивнее, чем Метрику). Метрика в свою очередь может попадать под блокировки за пределами рунета. Оба инструмента недосчитывают 15-30% трафика.

## Практическая рекомендация: используй оба

Это не дипломатичный ответ — это рабочий подход. Вот как это организовать:

### Метрика — как основной инструмент для рунета

- Вебвизор для UX-анализа и отладки
- Карты кликов для оптимизации страниц
- Интеграция с Яндекс.Директ для управления рекламой
- Отчёты для маркетологов, которые не пишут SQL

### GA4 — как источник сырых данных

- BigQuery-экспорт для построения собственной аналитики
- Кросс-платформенное отслеживание для мобильных приложений
- Predictive audiences для рекламных кампаний в Google
- Exploration reports для ad-hoc анализа

### Как не сойти с ума от двух систем

1. **Общий слой событий.** Используй GTM, чтобы один dataLayer отправлял данные в обе системы. Не дублируй код на сайте.

2. **Единый источник правды.** Выбери одну систему для финансовой отчётности. Обычно это GA4 + BigQuery, потому что данные сырые и проверяемые. Метрика — для оперативного анализа.

3. **Не сравнивай цифры напрямую.** Метрика и GA4 считают сессии по-разному, используют разные модели атрибуции, по-разному обрабатывают ботов. Расхождение в 10-15% — это норма. Сравнивай тренды, не абсолютные числа.

4. **Документируй настройки.** Один файл: какие цели настроены, где, с какими ID. Когда через полгода что-то сломается (а оно сломается), тебе будет где искать.

## Итого: быстрая шпаргалка

| Критерий | Метрика | GA4 |
|---|---|---|
| Порог входа | Низкий | Высокий |
| Запись сессий | Вебвизор | Нет (нужен Clarity/Hotjar) |
| Тепловые карты | Встроены | Нет |
| Сырые данные | Logs API | BigQuery (бесплатно) |
| ML-фичи | Минимум | Predictive audiences |
| Рунет-экосистема | Отлично | Слабо |
| Google-экосистема | Слабо | Отлично |
| Семплирование | Нет | Есть в бесплатной версии |
| Кросс-девайс | Базово | Продвинуто |

Нет «лучшего» инструмента. Есть инструмент, который решает твою задачу. А ещё лучше — два инструмента, каждый на своём месте. Не воюй за бренд, используй то, что даёт результат.
